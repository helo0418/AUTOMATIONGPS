# 🛰️ PRUEBA TÉCNICA - INGENIERO QA - HEIMY LÓPEZ

## 📌 Objetivo

Este repositorio contiene la solución a la prueba técnica de QA solicitada por **TechAVL**, cuyo objetivo es diseñar y ejecutar pruebas automatizadas para funcionalidades críticas de una plataforma de **seguimiento GPS en tiempo real**, validando componentes como frontend, backend y data layer.

---

## 🛠️ Herramientas Utilizadas

| Componente | Herramienta        | Justificación |
|------------|--------------------|----------------|
| Frontend (React) | Cypress             | Ideal para pruebas end-to-end en apps React, fácil de usar e integrar. |
| Backend (Golang/API) | Postman + Newman  | Permiten pruebas funcionales y automatizadas de APIs REST. |
| Data Layer (Kafka + Cassandra) | Python, kafka-python, cassandra-driver | Facilitan simulación de flujos de datos GPS en tiempo real. |
| Pruebas de carga (extra) | K6                 | Permite evaluar rendimiento bajo usuarios concurrentes. |
